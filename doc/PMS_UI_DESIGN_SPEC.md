# 宝寓类 PMS 系统详细 UI/UX 设计规范书

| 文档类型 | UI 设计规范 |
| :--- | :--- |
| **版本** | V1.0 |
| **设计原则** | 移动端优先 (Mobile First)、高信噪比、操作容错性强 |
| **适用终端** | 房东端 (App/小程序), 住客端 (微信小程序) |

---

## 1. 房东端核心界面 (B端)

### 1.1 房态日历页 (首页核心)
这是房东每天打开频率最高的页面，要求信息密度高但布局清晰。

*   **顶部导航栏 (Header)**:
    *   **左侧**: 当前月份/年份 (点击可切换日期)。
    *   **右侧**: 
        *   `筛选`图标: 按房型分组/按楼层分组筛选。
        *   `搜索`图标: 输入姓名/手机号查找订单。
        *   `+` 号按钮: 快速创建“线下订单”或“锁房”。
*   **次级状态栏**:
    *   显示图例小点: ⚫️已入住 🔴已预订 ⚪️空房 🔒维修/锁房。
    *   显示“回到今天”按钮。
*   **日历主体 (Grid View)**:
    *   **Y轴 (纵向)**: 房间列表 (图片头像 + 房号 + 房型名)。
    *   **X轴 (横向)**: 日期 (显示周几，周末日期高亮显示)。
    *   **订单色块**: 
        *   横跨日期的长条矩形。
        *   **颜色**: 区分渠道 (如 Airbnb=粉红, 携程=深蓝, 美团=黄色, 线下=绿色)。
        *   **内容**: 显示“住客姓名” + “￥价格” + “渠道Logo小标”。
        *   **状态标识**: 右上角小三角表示状态 (入住中/未支付/已离店)。
*   **底部浮动操作栏 (FAB)**:
    *   **批量操作**: 点击后，日历进入多选模式，可框选多个日期的房间进行“批量改价”或“批量关房”。

### 1.2 订单详情页 (Order Detail)
*   **状态卡片 (Top Card)**:
    *   **背景色**: 根据状态变化 (待入住=橙色, 在住=绿色, 已离店=灰色)。
    *   **主标题**: 大字显示状态 (如 "待入住")。
    *   **倒计时**: "距离入住还有 2 天"。
    *   **核心操作区**: `办理入住`、`发入住指南`、`修改订单`。
*   **住客信息区**:
    *   **联系人**: 姓名 + 手机号 (右侧带`拨号`和`复制`图标)。
    *   **来源**: 渠道 Logo + 外部订单号 (点击复制)。
*   **入离信息区**:
    *   **房间**: 房型名称 (点击跳转房型详情)。
    *   **时间**: 入住日期 (周几) 14:00 -> 离店日期 (周几) 12:00 (共 X 晚)。
*   **费用明细区 (折叠面板)**:
    *   **总价**: 大号字体。
    *   **明细**: 房费 x 晚、清洁费、押金 (高亮显示押金状态: 未付/已冻结/已退)。
*   **底部操作栏**:
    *   `取消订单` (二次确认弹窗)。
    *   `退房` (关联触发保洁任务)。

### 1.3 渠道直连配置页 (Channel Config)
*   **列表页**:
    *   卡片式列表展示各大 OTA (Airbnb, 携程, 美团, 途家)。
    *   **状态开关**: 右侧 Switch 开关 (已连接/未连接)。
    *   **状态灯**: 🟢 正常同步中 | 🔴 授权过期/异常。
*   **映射详情页 (Mapping)**:
    *   **账号信息**: 显示当前绑定的 OTA 账号昵称。
    *   **房源映射表**: 
        *   左列: PMS 本地房型 (固定)。
        *   右列: 下拉选择框 (读取到的 OTA 远程房型)。
        *   *交互*: 选择对应关系后，显示“✅ 已匹配”。
    *   **同步策略开关**:
        *   [x] 同步库存 (PMS变动 -> OTA)。
        *   [x] 同步价格 (PMS变动 -> OTA)。
        *   [x] 自动落单 (OTA订单 -> PMS)。

### 1.4 经营看板 (Dashboard)
*   **数据概览 (卡片组)**:
    *   今日入住 (数) | 今日离店 (数) | 剩余空房 (数)。
    *   本月营收 (金额) | 实时入住率 (%)。
*   **图表区**:
    *   **近7日营收趋势**: 折线图。
    *   **渠道占比**: 环形图 (Airbnb 40%, 美团 30%, ...)。
*   **待办事项 (To-Do List)**:
    *   🔴 3 个未处理订单。
    *   🟠 2 个待打扫房间。
    *   🟡 1 个明日到期 OTA 授权。

## 2. 住客端/微店界面 (C端)

### 2.1 微店首页 (Home)
*   **Hero Banner**: 全屏大图轮播 (展示民宿最美照片)。
*   **搜索悬浮框**:
    *   入住日期 / 离店日期 (点击弹出日历选择器)。
    *   `搜索房源` 按钮 (品牌色背景)。
*   **房源列表 (Feed流)**:
    *   **大卡片布局**:
        *   上部: 房间封面图 (支持左右滑动查看更多图)。
        *   下部: 
            *   **标题**: 房间名 (如 "海景落地窗大床房")。
            *   **标签**: 🏷️近地铁 🏷️投影仪 🏷️可做饭。
            *   **价格**: ￥XXX/晚 (划线价: ￥YYY)。
            *   **预订按钮**: 仅剩X间 / 满房 (置灰)。

### 2.2 预订确认页 (Booking)
*   **房源信息简报**: 小图 + 房名 + 日期段。
*   **入住人表单**:
    *   姓名 (自动填入微信昵称/上次填写)。
    *   手机号 (一键获取微信手机号)。
    *   身份证 (OCR 拍照识别)。
*   **优惠券**: 点击选择可用优惠券。
*   **底部支付栏**:
    *   左侧: "合计 ￥XXX"。
    *   右侧: `微信支付` 按钮。

### 2.3 入住指南页 (Check-in Guide)
*   *触发场景*: 订单支付成功后，或入住当天早上推送。
*   **地图导航**: 嵌入地图组件，显示“一键导航”按钮。
*   **门锁密码**: 
    *   大字号显示密码 (如 `8829#`)。
    *   *动态效果*: 仅在入住时间内显示，未到时间显示“入住当天 12:00 解锁”。
*   **Wi-Fi 卡片**: 点击一键复制密码。
*   **图文指引**: "如何打开热水器", "投影仪使用说明" (折叠式图文)。

## 3. 交互细节说明

1.  **加载状态**:
    *   日历加载时，显示骨架屏 (Skeleton Screen)，避免页面跳动。
2.  **错误处理**:
    *   渠道同步失败时，顶部弹出红色 Toast 提示，并伴有震动反馈。
3.  **手势操作**:
    *   日历页支持双指缩放 (Zoom) 以调整网格密度。
    *   订单列表支持左滑 (Swipe) 快速操作 (如“备注”, “标记已读”)。
4.  **色彩规范**:
    *   **品牌色**: #00BFA5 (青绿色，代表舒适、安全)。
    *   **警告色**: #FF5252 (用于满房、冲突、未支付)。
    *   **链接色**: #2979FF (用于电话号码、网址)。
